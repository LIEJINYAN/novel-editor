## 简介

NovelEngine Editor 是一款现代小说编辑器，结合了富文本编辑（Tiptap）和代码编辑（Monaco），并提供 AI 辅助写作功能。支持桌面端（Tauri）和 Web 端。

### 主要特性

- ✍️ 富文本编辑器（Tiptap + ProseMirror）
- 🤖 AI 辅助写作（24 种 AI 命令 + 6 个模板）
- 📑 多标签页编辑
- 🎯 写作模式（小说/散文/诗歌/笔记/专注）
- 🔍 全文搜索与替换
- 📊 写作统计与目标追踪
- 🌙 暗色/亮色主题
- 🌐 中英文切换
- ☁️ 云同步（WebDAV/GitHub/Gitee）
- 🧩 插件系统
- 📱 移动端适配

## 安装与启动

### Web 端

1. 直接访问部署地址
2. 或本地运行：
   ```bash
   pnpm install
   pnpm dev
   ```

### 桌面端（Tauri）

1. 安装 Rust 和 Tauri CLI
2. 构建桌面应用：
   ```bash
   pnpm install
   pnpm tauri:build
   ```
3. 安装包位于 `src-tauri/target/release/bundle/`

## 文档管理

### 创建文档

- 点击左侧边栏的 `+` 按钮
- 或使用快捷键 `Ctrl+N`

### 重命名文档

- 右键点击文档，选择"重命名"
- 或双击文档名称

### 删除文档

- 右键点击文档，选择"删除"

### 拖拽排序

- 拖拽文档可调整顺序

## 富文本编辑

### 基本格式

| 功能 | 快捷键 | 按钮 |
|------|--------|------|
| 加粗 | Ctrl+B | **B** |
| 斜体 | Ctrl+I | *I* |
| 下划线 | Ctrl+U | U |
| 删除线 | Ctrl+Shift+S | S |
| 代码 | Ctrl+E | `</>` |
| 高亮 | Ctrl+Shift+H | 🖍️ |

### 标题

- 使用 `#` 到 `######` 创建标题
- 或使用工具栏的标题下拉菜单

### 列表

- 无序列表：点击工具栏按钮或输入 `- `
- 有序列表：点击工具栏按钮或输入 `1. `
- 任务列表：点击工具栏按钮

### 引用

- 输入 `>` 创建引用块
- 或使用工具栏按钮

### 代码块

- 输入 ``` 创建代码块
- 支持语法高亮

## 多标签页

### 打开标签页

- 点击文档自动打开新标签页
- 最多同时打开 10 个标签页

### 切换标签页

- 点击标签页
- 使用 `Ctrl+1` 到 `Ctrl+9` 快速切换
- 使用 `Ctrl+Tab` 或 `Ctrl+Shift+Tab` 循环切换

### 关闭标签页

- 点击标签页的 × 按钮
- 或使用 `Ctrl+W`

### 文档图标

- 点击标签页左侧的图标可切换图标
- 支持 7 种图标：📝📖📄🎭📋📊🗂️

## 专注模式

### 进入专注模式

- 使用 `Ctrl+Shift+F`
- 或点击状态栏的专注模式按钮

### 专注模式功能

- 隐藏所有 UI 元素
- 仅显示编辑区域
- 支持右键菜单
- 边缘触发工具栏（鼠标移到顶部）
- `ESC` 退出专注模式

### 配置

在设置中可配置：
- 自动隐藏工具栏
- 始终显示工具栏
- 从不显示工具栏

## 云同步

### WebDAV

1. 打开设置 → 云同步
2. 选择 WebDAV
3. 输入服务器地址、用户名、密码
4. 点击测试连接
5. 保存配置

### GitHub

1. 打开设置 → 云同步
2. 选择 GitHub
3. 输入 Personal Access Token
4. 输入仓库名称
5. 点击测试连接
6. 保存配置

### Gitee

1. 打开设置 → 云同步
2. 选择 Gitee
3. 输入 Access Token
4. 输入仓库名称
5. 点击测试连接
6. 保存配置

## 常见问题

### Q: 如何备份数据？

A: 数据保存在浏览器 IndexedDB 中。可以使用云同步功能备份到 WebDAV/GitHub/Gitee。

### Q: 如何恢复数据？

A: 使用云同步功能从云端恢复数据。

### Q: 支持哪些浏览器？

A: 支持 Chrome、Firefox、Edge、Safari 等现代浏览器。

### Q: 如何更新编辑器？

A: Web 端自动更新。桌面端需要重新下载安装包。

### Q: AI 功能需要付费吗？

A: AI 功能使用 OpenAI 兼容接口，需要自行配置 API Key。

### Q: 如何反馈问题？

A: 请访问 GitHub Issues 页面反馈问题。
